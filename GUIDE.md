## 🏗 Как обновлять сайт (Деплой)
Чтобы изменения из кода перешли на живой сайт, всегда используй эту цепочку команд:

1. **Скачать обновления:**
   ```bash
   git pull origin main
   ```
2. **Собрать проект (самое важное!):**
   ```bash
   npm run build
   ```
3. **Перезапустить сервер:**
   ```bash
   pm2 restart all
   ```
   *Если `restart` не помог, используй жесткий перезапуск:*
   ```bash
   pm2 delete all && pm2 start npm --name "knol" -- start
   ```

## 🛠 Решение частых проблем

### 1. "Application error: a client-side exception..."
Если сайт выдает белую страницу с такой ошибкой после обновления:
*   Нужно очистить кеш сборки и пересобраться:
    ```bash
    rm -rf .next
    npm run build
    pm2 restart all
    ```

### 2. Дизайн не обновился (хотя код на сервере есть)
*   **Сбрось кеш браузера:** Нажми `Ctrl + F5` на сайте.
*   **Режим Инкогнито:** Проверь через `Ctrl + Shift + N`.
*   **Проверка кода:** Введи `grep "нужное_слово" путь/к/файлу`, чтобы убедиться, что файл на сервере действительно содержит правку.

## 🤖 Подсказки для ИИ (Промпты)

Используй эти термины, чтобы нейросеть понимала тебя с полуслова при правке интерфейса:

*   **«Убери мерцание/фликкер (flickering)»** — когда экран мигает при переключении страниц или открытии окон.
*   **«Сделай точечное обновление (Partial Update)»** — проси это, когда не хочешь, чтобы страница перезагружалась целиком, а менялся только один элемент.
*   **«Не сбрасывай innerHTML во время анимации»** — техническая просьба, если анимация прерывается или «дергается».
*   **«Вынеси стили из JS в CSS»** — если шрифты или размеры элементов «скачут» в первую секунду загрузки.

**💡 Идеальный рецепт промпта:**
> «При клике на [элемент] происходит мерцание. Сделай обновление бесшовным (точечным), чтобы каркас страницы не перерисовывался целиком».
