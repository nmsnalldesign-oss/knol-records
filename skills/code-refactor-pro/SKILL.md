Этот скилл активируется, когда пользователь просит "улучшить", "отрефакторить" или "сделать код чище".

## Инструкции по работе:
1. **Анализ запахов**: Найди в коде дублирование, слишком длинные функции, "божественные" объекты или жесткие зависимости.
2. **Применение SOLID**:
   - S: Разбей крупные классы на узкоспециализированные.
   - O: Используй полиморфизм вместо бесконечных `switch/case`.
   - D: Внедряй инверсию зависимостей.
3. **Безопасность**: При рефакторинге следи, чтобы логика не изменилась. Всегда проверяй сайд-эффекты.
4. **Документирование**: Кратко объясни, почему предложенный вариант лучше оригинала.

## Алгоритм:
- Сначала просмотри весь файл.
- Предложи план изменений.
- Реализуй изменения по одному логическому блоку за раз.
