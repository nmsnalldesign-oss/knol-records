# Модуль 4: Автоматизация с n8n

## Что такое n8n?

**n8n** (произносится "н-эйт-н") — это инструмент автоматизации workflows с открытым исходным кодом. Позволяет соединять разные сервисы и автоматизировать задачи БЕЗ написания кода.

### Простыми словами

Представь конвейер:
```
[Триггер] --> [Действие 1] --> [Действие 2] --> [Результат]

Пример:
[Новый заказ] --> [Отправить email] --> [Добавить в таблицу] --> [Уведомить в Telegram]
```

### Зачем это нужно?

| Без автоматизации | С n8n |
|-------------------|-------|
| Вручную проверяешь оплаты | n8n проверяет автоматически |
| Вручную отправляешь файлы | n8n отправляет сразу после оплаты |
| Вручную ведёшь базу клиентов | n8n добавляет данные автоматически |

## Как начать с n8n

### Способ 1: Облачная версия (проще)

1. Зайди на https://n8n.io/
2. Нажми "Get started free"
3. Создай аккаунт
4. Получишь бесплатный план с 5 workflows

### Способ 2: Локальная установка (бесплатно без ограничений)

```powershell
# Установка через npm
npm install n8n -g

# Запуск
n8n start

# Открой в браузере: http://localhost:5678
```

### Способ 3: Docker (рекомендуется для продакшена)

```powershell
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  n8nio/n8n
```

## Основные концепции n8n

### 1. Node (Узел)

Каждый блок в workflow называется **узлом**. Узлы бывают:

- **Trigger** — запускает workflow (по времени, по webhook, по событию)
- **Action** — выполняет действие (отправить email, записать в БД)
- **Logic** — управляет потоком (условия, циклы, объединение)

### 2. Workflow (Рабочий процесс)

Цепочка узлов, соединённых вместе.

```
[Webhook] --> [IF условие] --> [Да: Отправить email]
                           --> [Нет: Записать лог]
```

### 3. Credentials (Учётные данные)

Для подключения к сервисам (Telegram, Google, Email) нужны API-ключи. 
n8n хранит их безопасно и позволяет переиспользовать.

## Твой первый Workflow

### Задача: Уведомление в Telegram при заполнении формы

```
[Webhook] --> [Отправить в Telegram]
```

### Пошаговая инструкция

**Шаг 1: Создай webhook**
1. Добавь узел "Webhook"
2. Метод: POST
3. Скопируй URL вебхука

**Шаг 2: Добавь Telegram**
1. Добавь узел "Telegram"
2. Создай бота через @BotFather
3. Получи API Token
4. Укажи Chat ID

**Шаг 3: Соедини узлы**
1. Перетащи линию от Webhook к Telegram
2. В Telegram настрой сообщение:
   ```
   Новая заявка!
   Имя: {{ $json.name }}
   Email: {{ $json.email }}
   ```

**Шаг 4: Активируй и протестируй**
1. Нажми "Activate"
2. Отправь POST-запрос на webhook URL

## Полезные узлы n8n

| Узел | Назначение |
|------|------------|
| **Webhook** | Принимает HTTP-запросы |
| **Schedule Trigger** | Запуск по расписанию (каждый час, день) |
| **HTTP Request** | Отправляет запросы к API |
| **Google Sheets** | Работа с гугл-таблицами |
| **Telegram** | Отправка сообщений в Telegram |
| **Email (SMTP)** | Отправка писем |
| **IF** | Условная логика |
| **Set** | Установка переменных |
| **Code** | Свой JavaScript/Python код |

## Интеграция n8n с твоим сайтом

### Сценарий: Автоматическая доставка после оплаты

```
[Продамус webhook] --> [Проверить статус] --> [Отправить email с файлом]
                                          --> [Добавить в Google Sheets]
                                          --> [Уведомить в Telegram]
```

### Пример workflow

1. **Webhook от Продамус** — получает данные об оплате
2. **IF узел** — проверяет `payment_status === 'success'`
3. **Email узел** — отправляет письмо с ссылкой на материалы
4. **Google Sheets** — добавляет запись о покупателе
5. **Telegram** — уведомляет тебя о продаже

## Практическое задание

Создай workflow:
1. Webhook принимает данные формы (имя, email, сообщение)
2. Отправляет тебе уведомление в Telegram
3. Сохраняет данные в Google Sheets
4. Отправляет автоответ клиенту на email

**Подсказка:** Начни с простого (только webhook + telegram), потом добавляй остальное.
