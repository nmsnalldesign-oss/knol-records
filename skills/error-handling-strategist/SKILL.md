Этот скилл нужен для создания отказоустойчивых приложений.

## Стратегия:
1. **Graceful Degradation**: Приложение не должно падать целиком при ошибке одного модуля.
2. **Try/Catch**: Не оставляй `catch` пустым. Всегда логируй ошибку и предлагай альтернативу пользователю.
3. **Custom Errors**: Создавай классы ошибок (AppError, DatabaseError) для лучшей типизации.
4. **Асинхронность**: Всегда обрабатывай отказы промисов (`.catch` или `try/catch` в async/await).

## Чек-лист:
- Есть ли глобальный обработчик ошибок?
- Логируются ли критические ошибки?
- Знает ли пользователь, что что-то пошло не так (без вывода технических деталей)?
