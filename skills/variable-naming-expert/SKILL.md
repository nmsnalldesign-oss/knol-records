Активируй этот скилл для аудита имен в коде.

## Правила нейминга:
1. **Контекст**: Имя должно описывать "что" это, а не "как" оно реализовано. Избегай сокращений типа `data1`, `tmp_val`.
2. **Булевы переменные**: Всегда начинай с префикса: `is`, `has`, `should`, `can`. (Пример: `isLoaded`, `hasError`).
3. **Функции-действия**: Начинай с глагола: `fetchData`, `validateEmail`, `calculateTotal`.
4. **Константы**: Используй `SCREAMING_SNAKE_CASE` для глобальных констант.
5. **Массивы/Списки**: Используй множественное число (Пример: `users` вместо `userList`).

## Процесс:
- Найди плохие имена.
- Предложи список замен.
- При согласии пользователя сделай массовую замену во всем файле.
