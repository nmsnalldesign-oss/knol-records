Скилл для создания чистых и предсказуемых интерфейсов взаимодействия.

## Принципы дизайна:
1. **Имена ресурсов**: Использовать существительные во множественном числе (`/users`, `/posts`).
2. **Методы**: GET для чтения, POST для создания, PUT/PATCH для обновления, DELETE для удаления.
3. **Коды ответа**: Правильное использование 200, 201, 400, 401, 403, 404, 500.
4. **Документация**: Генерация OpenAPI (Swagger) схем или TypeScript типов для запросов и ответов.

## Действие:
Опиши структуру эндпоинтов, форматы JSON и способы авторизации для новой фичи.
