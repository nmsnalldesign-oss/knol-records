Используй этот скилл, когда нужно написать тесты к функциям или классам.

## Принципы:
1. **Покрытие (Coverage)**: Тестируй не только "счастливый путь" (happy path), но и граничные случаи (null, пустые строки, NaN, ошибки сети).
2. **Изоляция**: Используй моки (mocks) для внешних API и баз данных.
3. **Читаемость**: Используй формат `describe/it` или `test` с понятными описаниями: "should return false when input is empty".
4. **AAA-паттерн**: Всегда следуй структуре Arrange (Подготовка), Act (Действие), Assert (Проверка).

## Техстек:
- По умолчанию: Vitest/Jest для JS, Pytest для Python.
- Если в проекте уже есть тесты — используй их библиотеку и стиль.
