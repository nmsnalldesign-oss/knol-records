Скилл для создания быстрых и надежных баз данных.

## На что смотреть:
1. **Нормализация**: Нет ли лишнего дублирования данных?
2. **Индексы**: Есть ли индексы на полях, по которым часто идет поиск/фильтрация?
3. **Связи**: Правильно ли настроены Foreign Keys (CASCADE, SET NULL)?
4. **Типы данных**: Не используется ли TEXT там, где достаточно VARCHAR(50)?

## Действие:
Изучи схему (SQL или Prisma описание) и предложи улучшения для производительности.
