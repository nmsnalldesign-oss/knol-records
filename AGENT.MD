# Правила разработки (System Instructions)
1. **Стек технологий:** Next.js (App Router), React, Tailwind CSS для стилей, Zustand для стейт-менеджмента. Для базы данных и админки — Supabase (или локальный SQLite для старта).
2. **Модульность:** Никогда не переписывай весь файл, если нужно изменить одну кнопку. Выноси логику в изолированные компоненты (например, `TrackCard.tsx`, `AudioPlayer.tsx`).
3. **Безопасность:** Админ-панель должна быть закрыта роутингом. Дефолтные креды (admin/admin) используем только на этапе разработки в `.env.local` файле.
4. **Запрет на регрессию:** Перед изменением существующего компонента, убедись, что новые изменения не сломают текущий функционал плеера или маршрутизации.
